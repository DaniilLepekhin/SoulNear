fix: resolve env variables loading and ContainerConfig error

## Problem
- Docker Compose was trying to interpolate ${VAR} from HOST environment (empty)
- This was overwriting values from .env.prod file
- Result: "WARNING: The POSTGRES_PASSWORD variable is not set"
- Also: "KeyError: 'ContainerConfig'" when recreating containers

## Solution
1. **docker-compose.yml**: Removed all ${VAR} interpolations from environment sections
   - Only kept Docker-specific overrides (POSTGRES_HOST, POSTGRES_PORT)
   - All other variables now loaded via env_file: .env.prod

2. **Makefile**: Removed duplicate --env-file flags from commands
   - env_file already specified in docker-compose.yml

3. **validate-env.sh**: Now always checks .env.prod (hardcoded in compose)

4. **scripts/safe_redeploy.sh**: Created safe redeploy script WITHOUT losing DB
   - Stops containers (keeps volumes)
   - Removes old container metadata
   - Rebuilds and starts

## Single Source of Truth
All environment variables now live in ONE place: .env.prod (next to docker-compose.yml)
