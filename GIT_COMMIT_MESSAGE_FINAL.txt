fix: correct async execution_options usage in database creation

## Problem
Previous fix attempted to use execution_options on connection:
```python
await conn.execution_options(isolation_level='AUTOCOMMIT').execute(...)
```

This failed with: `AttributeError: 'coroutine' object has no attribute 'execute'`

## Root Cause
In async SQLAlchemy:
- `execution_options()` returns a coroutine when called on connection
- Even if awaited, it returns a new connection object
- The correct approach is to set execution_options on ENGINE, not connection

## Solution
Set isolation_level on engine before creating connection:

**Before (broken):**
```python
async with admin_engine.connect() as conn:
    await conn.execution_options(isolation_level='AUTOCOMMIT').execute(...)
```

**After (correct):**
```python
admin_engine = _build_engine('postgres').execution_options(
    isolation_level='AUTOCOMMIT'
)
async with admin_engine.connect() as conn:
    await conn.execute(text(f'CREATE DATABASE "{POSTGRES_DB}"'))
```

## Technical Details
- `execution_options()` on engine is synchronous (no await needed)
- Returns a new engine with modified settings
- All connections from that engine inherit the isolation level
- CREATE DATABASE now executes outside transaction ✅
